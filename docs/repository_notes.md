# Repository Notes

## Layout decision

This repository intentionally preserves the Posit/Shiny app layout. Unlike the companion analysis repository, files are not reorganized into `code/` and `data/` folders because `app.R` reads app data from root-level files and the `data_arrow/` directory.

## Deployment-critical files

- `app.R`: main Shiny application.
- `manifest.json`: Posit deployment manifest.
- `requirements.txt`: Python helper requirements generated by `rsconnect-python`.
- `data_arrow/`: Arrow/Feather data files reconstructed into `SummarizedExperiment` objects at runtime.
- `secondary_data.RData`: secondary app data, including MOFA objects.
- `www/mammoth_icon2.png`: UI icon.

## Cleanup performed

Non-app artifacts were removed from version control:

- `.DS_Store`
- `.RDataTmp`
- `.Rapp.history`

These were not referenced by `app.R` or `manifest.json`.

## Publication release checklist

- Confirm final manuscript title and author order.
- Confirm GEO accession numbers and add them to `README.md`.
- Confirm code/data licensing with the PI/lab before making the repository public.
- After any Posit redeployment, commit updated `manifest.json` if it changes.
- Create a tagged release corresponding to the accepted manuscript.
